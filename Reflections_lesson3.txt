When would you create a remote repo rather than keeping all of your work offline?

Backup, Sync between devices, coop with other programmers.

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date with your remote repository?
If the changes are an experiment.

Describe the differences between forks, clones, and branches. When would you use one instead of another?

forking is used when you want to edit a repo but the author needs the (unchanged) original. CLoning is the same, but offline.
Branches are used for adding experimental new features or differemt versions.
